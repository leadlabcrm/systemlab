# Session 09 - May 21, 2026

## Objective
Fix the broken login (no response on "Enter Workspace" click), integrate Firebase RTDB for cloud sync, and fix PDF flowchart rendering.

## Root Cause Identified
**Critical Bug**: Inside `printDoc()`, the PDF template literal contained literal `<script>` and `</script>` HTML tags. The browser's HTML parser saw `</script>` as the closing tag for the main script block, which caused ALL JavaScript defined after line ~1346 (including `loginApp`, `bootApp`, `connectToWorkspace`, `persist`, and hundreds of other functions) to be completely ignored. This is why clicking "Enter Workspace" had zero response.

## Changes Made

### 1. Fixed the Critical Script Tag Bug
- Inside `printDoc()` template literal, changed `<script>` to `<${'script'}>` so HTML parser doesn't see it as a literal script tag.
- This one change unblocked the entire application.

### 2. Firebase Backend Migration
- Removed Firestore dependency (required security rules that couldn't be deployed without auth).
- Removed Firebase Anonymous Auth approach.
- Switched to **Firebase Realtime Database (RTDB) REST API** — uses plain `fetch()`, no SDK needed.
- RTDB URL confirmed: `https://system-labs-default-rtdb.firebaseio.com`
- Nithin created the RTDB in Firebase Console (test mode = public rules).

### 3. Graceful Login Architecture
- Login is now **instant and offline-first** — app loads from `localStorage` immediately.
- Cloud sync via RTDB runs in the background after login.
- If RTDB is unreachable, app still works in local mode.
- `"☁️ Cloud sync active"` toast appears when cloud connects successfully.
- Each workspace is stored under `workspaces/<passcode>` in RTDB.
- Workspace data also cached in `localStorage` under key `systemlab_ws_<workspaceId>`.

### 4. PDF Export Improvements
- Flowchart SVG nodes forced to white-fill/black-stroke for clean print output.
- Print opens in a dedicated new window (bypasses app layout constraints).

### 5. Removed Duplicate persist() Function
- The old `persist()` (line ~811) that only saved to localStorage was removed.
- New `persist()` saves to both localStorage and RTDB (if cloud is active).

## Addendum 2 (Click-to-Edit Scroll Sync)
Implemented "Scroll Sync" / "Source Mapping" in the editor.
- **How it works**: Hovering over any section in the live Markdown preview (e.g. a paragraph, heading, list, or flowchart) now shows a subtle orange highlight. Clicking that section instantly moves the cursor and scrolls the raw Markdown editor on the left to the exact line of code that generated it.
- **Implementation**: The custom `mdRender` parser was upgraded to inject original line numbers as `data-line` attributes into the generated HTML. A delegated event listener handles clicks on `.sync-block` elements to calculate character offsets in the `<textarea>` and jump the cursor using `setSelectionRange()`.

## Addendum 3 (Versioning and Preview Full Screen)
- **Unified Save Logic**: Removed the redundant "Save Version" button. The primary "Save SOP" button (`saveDoc()`) now automatically creates a new version snapshot if the document's body has fundamentally changed since the last snapshot.
- **Fixed Version Restore**: The `restoreVer()` function was bugged because it called the old `saveNewVersion()` instead of properly updating the current document body. It now properly overwrites the text area and calls `saveDoc()` to capture the restoration as a new snapshot.
- **Full Screen Preview**: Added a `⛶ Full Screen` toggle button to the top-right of the Preview pane. Clicking it hides the markdown editor entirely, letting the live rendered preview take up 100% of the screen width for easier reading.
